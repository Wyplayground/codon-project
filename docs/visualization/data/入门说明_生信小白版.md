# `频率表` 文件夹是干什么的？（给刚开始学生信的同学）

这个文件夹里放的是：**黑曲霉（*Aspergillus niger*）在各种公开数据库里，64 种密码子各自用了多少次、占多少比例**。  
你可以把它理解成一张「**这本生物的 DNA 里，每种三联密码子（3 个字母一组）有多常见**」的统计表。

下面分三块说：**先懂概念 → 再看每个文件 → 你该怎么用**。

---

## 一、先懂 3 个概念（不用背，有个印象就行）

### 1. 什么是「密码子」？

DNA 由 **A、T、G、C** 四种字母组成。基因里编码蛋白质的部分，一般是 **3 个字母一组**，这一组就叫一个 **密码子（codon）**。  
例如 `ATG`、`GCC`、`TAA` 都是密码子（这里按 **DNA** 写；有些教材用 RNA 会把 T 写成 U，一回事）。

- 大多数密码子对应 **一种氨基酸**（蛋白质的积木）。
- 还有 **终止密码子**（告诉翻译机器「到这里蛋白结束」），例如 `TAA`、`TAG`、`TGA`。

一共 **4×4×4 = 64** 种密码子。

### 2. 什么是「密码子频率表」？

对很多条基因（很多段 DNA）统计：  
**每种密码子一共出现了多少次？**  
再除以总密码子数，得到 **比例（fraction）**，有时也写成 **「每千个密码子里有多少个」**（per_thousand）。

**为什么要关心？**  
不同物种、甚至同一物种不同菌株，偏好的密码子会不一样。做 **密码子优化**（把 DNA 改一改，蛋白不变，但希望在新宿主里更好表达）时，就要参考「宿主更常用哪些密码子」。

### 3. 为什么同一个物种会有好几张表？

因为 **统计口径不一样**：

| 类型 | 通俗理解 |
|------|----------|
| **合并物种表** | 把公共数据库里标成「黑曲霉」的很多基因凑在一起统计，像「全国抽样」。 |
| **某一个基因组组装** | 只数「这一版参考基因组」里注释出来的 CDS，像「按一本标准教科书统计」。 |

所以：**几张表数值接近是正常的，但不会完全一样**——不是谁错了，是**题目不一样**。

## 二、这个文件夹里每个文件是干什么的？

下面按 **建议阅读顺序** 排列。

### 1. 建议先看：`README_sources.md`

- **作用**：用表格列出每个数据文件**从哪来的**、重要链接（Kazusa、NCBI FTP 等）。
- **谁该看**：想快速知道「这些表分别对应哪个数据库」时打开它。

---

### 2. 四张「主数据表」：`*.tsv`（用 Excel / WPS / 记事本都能打开）

它们结构一样，都是 **很多行、几列**。每一行是一个密码子（如 `ATG`），列里是这个密码子在**该统计口径下**的计数和比例。

**共同列（看表头就行）：**

| 列名 | 初学者怎么理解 |
|------|------------------|
| `codon` | 三个字母的 DNA 密码子，大写。 |
| `count` | 这个密码子被统计了多少次（整数）。 |
| `fraction` | 占全部统计到的密码子的比例（0～1 之间的小数）。 |
| `per_thousand` | 每 1000 个密码子里大约有多少个是这个（和 fraction 是同一信息，换种写法）。 |
| `source` | 这一行数据属于哪个「来源标签」（脚本里写的名字）。 |
| `taxid` | NCBI 分类学 ID，黑曲霉这里是 **5061**。 |
| `assembly` | 若是「某个具体基因组版本」，这里会写 **GCF…** 或 **GCA…**；合并物种表可能是空的。 |
| `downloaded_on` | 脚本生成这条记录时的 UTC 时间。 |

下面四张表分别是什么：

#### （1）`aspergillus_niger_kazusa.tsv` —— Kazusa「物种合并」表

- **谁统计的**：日本 Kazusa 的密码子使用数据库，把公共库里很多「标注为黑曲霉」的编码序列汇总统计。
- **特点**：经典、好引用；是 **跨很多提交序列的合并**，不是某一个基因组文件一条条数出来的。
- **适合**：想跟文献里常见的「Kazusa 物种表」对齐时。

#### （2）`aspergillus_niger_refseq_cds_GCF_000002855.4_computed.tsv` —— NCBI RefSeq 重算表

- **谁统计的**：脚本下载 NCBI **RefSeq** 里指定版本 **`GCF_000002855.4`**（黑曲霉一个常用参考株背景）的 **`cds_from_genomic.fna.gz`**，在本地数密码子。
- **特点**：**可重复**——别人只要用同一版基因组文件，应能得到同样的数（过滤规则写在 `.meta.json`）。
- **适合**：你要写方法学里「我们按 RefSeq CDS 重算了密码子使用」时。

#### （3）`aspergillus_niger_ncbi_genbank_cds_GCA_000230395.2_computed.tsv` —— 另一个菌株/组装的 GenBank 重算表

- **谁统计的**：另一个常用组装 **`GCA_000230395.2`**（ATCC 1015 / ASPNI v3），同样是下载 CDS 再数密码子。
- **特点**：和上一张 **不是同一套 DNA 序列**，所以比例会有差异——像「另一本教材的用词习惯」。
- **适合**：你想看「不同黑曲霉参考基因组之间，密码子偏好差多少」。

#### （4）`aspergillus_niger_ncbi_hivecuts.tsv` —— 名字像 HIVE/CoCoPUTs，但要读说明

- **理想情况**：来自 FDA/GWU 那套 **HIVE-CUTs / CoCoPUTs** 网页工具导出的物种表。
- **你电脑上的实际情况**：自动化脚本去抓官网接口时，经常遇到 **403（禁止爬虫）**，所以脚本采用 **备用方案**：  
  **把 RefSeq 那张表（GCF_000002855.4）的计数原样复制过来**，并在元数据里写明这是 **「CoCoPUTs 方法论里 RefSeq 优先」的可复现替代**，不是你在浏览器里点出来的那份原始导出。
- **你怎么确认**：打开同名的 **`aspergillus_niger_ncbi_hivecuts.tsv.meta.json`**，看里面的 `note` 和 `proxy_of_tsv`。
- **若你必须和官网一模一样**：按 `meta.json` 里提示，去 FDA DNA HIVE 网页手动导出，再覆盖这个 `tsv` 文件。

---

### 3. 每张表旁边的 `*.meta.json` 是什么？

- **作用**：用 **JSON** 记录「这张表怎么来的」：下载链接、组装号、跳过了多少条序列、是否用了代理/快照等。
- **谁该看**：写实验记录、或怀疑「这张表是不是我想要的版本」时打开看。
- **不用怕**：就是几行说明文字，不会比 `tsv` 更难。

---

### 4. `cross_check_summary.tsv` —— 几张表互相有多像？

- **作用**：把几张表的 **`fraction` 列**拿来两两比较，算 **Pearson 相关系数**（越接近 1 越像），并列出差别最大的几个密码子。
- **谁该看**：你想一句话回答「Kazusa 和 RefSeq 差得多吗」时看这个。
- **怎么读**：每一行是两个表的名字 + 一个 `pearson_r` + 一串「差得最多的密码子」。

---

### 5. `fetch_log.txt` —— 脚本当时干了啥

- **作用**：文本日志，记录 Kazusa 是否在线成功、RefSeq/GenBank 下载是否成功、HIVE 是否 403、是否走了备用方案等。
- **谁该看**：出问题时给「会写脚本的同学」或发帖求助时附上。

---

### 6. `kazusa_raw.html` —— Kazusa 网页的原始备份（可选）

- **作用**：当次成功在线抓取 Kazusa 时，把返回的 **HTML 原文**存下来，方便核对「当时网页到底长什么样」。
- **谁该看**：一般不用；只有怀疑解析错了才需要。

---

### 7. `fetch_codon_tables.py` —— 生成上面所有东西的脚本

- **作用**：一键重新下载、重新计数、重新写 `tsv` / `meta` / `README` / `cross_check`。
- **谁该看**：你会跑一点 Python 命令行时再碰；初学者可以暂时当「黑盒子」。

---

## 三、初学者「最小使用路径」

1. **只想知道黑曲霉密码子大概偏好**：打开 **`aspergillus_niger_kazusa.tsv`**，按 `fraction` 从大到小排序，看哪些密码子最常见。  
2. **想跟「某一个参考基因组」严格一致**：优先 **`aspergillus_niger_refseq_cds_GCF_000002855.4_computed.tsv`**，并读它的 **`meta.json`**。  
3. **看到 `hivecuts` 不要慌**：先看 **`aspergillus_niger_ncbi_hivecuts.tsv.meta.json`** 里写的是「远程」还是「RefSeq 代理」。

---

## 四、和 CodonTransformer 的关系（一句）

**CodonTransformer 自带的 164 个宿主里没有黑曲霉。**  
这些表是给你 **自己做黑曲霉相关的密码子分析、或自己写优化规则** 用的，**不会自动让 CodonTransformer 变成黑曲霉模型**。
