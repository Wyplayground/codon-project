# 黑曲霉密码子候选评审简化版 HOWTO

适用文件：`index.html`

适用对象：湿实验人员、基因合成下单人员、需要比较 IDT / GenScript / JCat / 公司返回序列的人。

一句话说明：这个网页不是“自动证明哪条表达量最高”的工具，而是帮你检查多条 DNA 候选是否都编码同一个蛋白，并用黑曲霉密码子频率表做简洁比较。

---

## 1. 这个工具能做什么？

在不改变蛋白氨基酸序列的前提下，把多条候选 CDS 放在一起比较：

- 检查每条候选 DNA 是否能翻译成同一个目标蛋白。
- 拦截明显错误，例如长度不是 3 的倍数、含 `N`、内部终止密码子、非同义突变。
- 用黑曲霉（*Aspergillus niger*）Kazusa taxid 5061 密码子频率表计算三个核心指标：`CSI`、`GC%`、`CFD%`。
- 显示逐位点密码子对照图，方便查看每个氨基酸位置用了哪个密码子。
- 导出 CSV / JSON / FASTA，方便留档、讨论或发给合成公司。

本页完全离线运行，不会把序列上传到任何网站。

**（进阶）与仓库内频率表文件的关系**：同目录下的 [`aspergillus_niger_5061.csv`](aspergillus_niger_5061.csv) 为 Kazusa taxid **5061** 的原始计数表（列：`amino_acid`, `codon`, `observed_count`）；本页在浏览器内使用**与之相同数值**的内嵌表，并在载入时按氨基酸簇将计数归一化为相对频率，用于 CSI / CFD% 等。页面**不会**自动读取该 CSV；若 Kazusa 更新，请按 [`aspergillus_niger_5061_SOURCE.md`](aspergillus_niger_5061_SOURCE.md) 与 `generate_kazusa5061_table.py` 同步后替换内嵌块。

## 2. 使用前需要准备什么？

### 2.1 原始 / 目标序列

左侧第一栏“原始 linB / 目标序列”必须填写。

推荐输入原始 linB CDS：

```text
>linB_original
ATG...TAG
```

要求：

- 只包含 `A/C/G/T`。
- 长度最好是 3 的倍数。
- 推荐有起始密码子 `ATG` 和末端终止密码子 `TAA/TAG/TGA`。

如果只有蛋白序列，也可以输入氨基酸单字母序列。不过这样没有原始 DNA 作为参考，逐位点图里的 `original` 列会为空。

### 2.2 候选序列

左侧第二栏“候选序列”用于放湿实验人员、网站或基因公司给出的优化结果。

常见来源包括：

- 自己手动调整的序列。
- IDT 返回的序列。
- GenScript 返回的序列。
- JCat 优化结果。
- 基因合成公司给出的优化序列。

推荐用 FASTA 格式：

```text
>candidate_1 source=manual
ATG...TAA
>candidate_2 source=GenScript
ATG...TGA
>candidate_3 source=company_A
ATG...TAG
```

也可以用 CSV：

```csv
name,source,sequence
candidate_1,manual,ATG...
candidate_2,GenScript,ATG...
candidate_3,company_A,ATG...
```

CSV 中必须有 `sequence` 列；推荐同时保留 `name` 和 `source` 列。

### 2.3 Excel 文件怎么办？

这个离线 HTML 不直接读取 `.xlsx`。

如果候选在 Excel 里，请另存为 CSV，再导入。CSV 表头建议为：

```csv
name,source,sequence
```

## 3. 标准操作步骤

1. 双击打开 `index.html`。
2. 在“原始 linB / 目标序列”中粘贴原始 CDS，或点击文件按钮上传 FASTA / TXT。
3. 在“候选序列”中粘贴多条候选 FASTA / CSV，或上传候选文件。
4. 按需调整左侧「CFD 稀有密码子阈值」滑块（默认 0.30）；具体含义见页内「什么是 CFD 稀有密码子阈值？」说明。
5. 保持勾选“生成程序参考候选”，除非只想看外部候选。
6. 点击“分析候选”。
7. 依次查看“指标与排序”“逐位点密码子对照图”“逐位点明细表（导出用）”“导出”。

## 4. 结果怎么看？

### 4.1 指标与排序

这个页面最重要。

外部/人工候选排序只包含通过验证的候选。也就是说，候选必须满足：

- DNA 长度合法。
- 没有非法字符，例如 `N`。
- 没有内部终止密码子。
- 翻译后的蛋白与目标蛋白完全一致。

**状态（status）列**：外部候选显示为「通过」当且仅当 **`errors` 为空**且**翻译得到的蛋白序列与目标蛋白完全一致**（含长度与每一位氨基酸）。若存在任一条 `errors` 或蛋白不一致，则为「未通过」。**仅有 `warnings`（例如起始密码子不是 ATG）而 `errors` 为空时，仍可通过**；未通过的候选不会出现在本排序表中，而会列在下方「未通过的外部候选」。指标页可展开 **「CSI、GC%、CFD%、errors、warnings 与状态列说明」** 查看各列在本页中的定义与算式。

排序规则是：

1. `CSI` 越高越靠前。
2. 如果 `CSI` 接近，则 `CFD%` 越低越靠前。
3. 如果仍接近，则 `GC%` 越接近黑曲霉编码区参考 GC 值越靠前。

注意：这个排序只是辅助筛选，不等于表达量预测。最终表达量仍需湿实验验证。

### 4.2 原始序列基线

如果输入的是原始 CDS，页面会显示一行原始序列基线。

这行用于比较，不代表它参与外部候选排名。

### 4.3 未通过的外部候选

如果候选有问题，会出现在“未通过的外部候选”中。

常见错误解释：

| 错误 | 含义 | 建议 |
|---|---|---|
| 含非 A/C/G/T 碱基 | 序列里有 `N` 或其他字符 | 回到原始文件确认碱基 |
| 长度不是 3 的倍数 | 可能发生缺碱基、插入或复制不完整 | 检查是否漏复制开头/结尾 |
| 末端密码子不是终止密码子 | 候选没有 stop codon | 确认公司返回的是不是完整 CDS |
| 存在内部终止密码子 | CDS 中间出现 stop | 这条通常不能直接用于表达 |
| 翻译蛋白与目标不一致 | 出现非同义突变或移码 | 不应作为“同一蛋白”的密码子优化候选 |

### 4.4 程序生成参考候选

页面底部会显示 3 条本地生成的参考序列。

它们只作为参考，不参与总体排名。

| 名称 | 含义 |
|---|---|
| 最高频密码子版 | 每个氨基酸选择黑曲霉频率表中最高频的同义密码子，通常 CSI 较高，但可能让 GC/GC3 偏向单一 |
| 频率加权版 | 按黑曲霉同义密码子频率抽样生成，内部固定种子为 42，每次结果一致 |
| GC/GC3 平衡版 | 在相对适应度与黑曲霉编码区 GC、GC3 参考倾向之间折中；仅供对照，不保证表达量 |

这些序列可以作为讨论参考，但不建议不加判断地直接认为它们表达量最高。

## 5. 三个核心指标是什么意思？

### 5.1 CSI

`CSI` 可以理解为“这条 CDS 使用黑曲霉偏好密码子的程度”。

计算思路：

```text
某密码子的相对适应度 = 该密码子频率 / 同一氨基酸下最高频密码子的频率
CSI = 各编码密码子相对适应度的几何平均（本页跳过甲硫氨酸 M、色氨酸 W 与终止密码子）
```

一般来说，`CSI` 越高，说明越倾向使用黑曲霉常用密码子。

不要把 `CSI` 单独理解为表达量。它只是密码子使用偏好指标。

### 5.2 GC%

`GC%` 是整条 CDS 中 `G` 和 `C` 的比例。

过高或过低都可能影响合成、扩增、二级结构或表达，但它不是越高越好。

### 5.3 CFD% 与 CFD 稀有密码子阈值

先对每个编码密码子计算**相对适应度**：该密码子在黑曲霉 Kazusa 表中的频率 ÷ 同一氨基酸下频率最高的同义密码子频率（0～1）。左侧滑块设定 **CFD 稀有密码子阈值**：相对适应度**严格小于**该值的编码密码子计为「稀有」。**CFD%** = 稀有密码子个数 ÷ 参与统计的编码密码子个数 × 100%；**终止密码子不计入**分子与分母。阈值调高时，更多密码子会被判为稀有，`CFD%` 往往会升高；该阈值只影响 `CFD%` 与逐位点图中的「稀有」着色，**不改变 CSI 与 GC% 的计算**。默认常用 0.30。

文献或工具里「CFD」可能有不同定义；本页与逐位点明细中的 `rare` 列一致。

一般来说，`CFD%` 越低，说明被判为低适配的密码子越少。

### 5.4 指标表中的 `errors` 与 `warnings`

- **`errors`**：硬错误列表，例如含非 A/C/G/T 碱基、长度不是 3 的倍数、序列过短、存在内部终止密码子、翻译含未知密码子、翻译蛋白与目标不一致、外部候选缺少终止密码子等。只要存在任一条，该候选即**未通过**。
- **`warnings`**：提醒信息，例如起始密码子不是 ATG、或（仅对原始基线）末端不是终止密码子等。**仅有 warnings 而 errors 为空时，外部候选仍可通过验证。**

## 6. 逐位点密码子对照图怎么看？

“逐位点密码子对照图”是一张按蛋白位置展开的表。

- 每一行：一个氨基酸位置，也就是一个密码子位置。
- 每一列：一条序列，例如原始序列、某条外部候选、本地参考候选。
- 每个格子：该序列在这个位置使用的 3 bp 密码子，例如 `GCT`、`GCC`。

颜色含义：

| 颜色/状态 | 含义 |
|---|---|
| 与原始序列相同 | 候选密码子等于原始 linB 对应密码子 |
| 同义替换 | 密码子变了，但翻译出的氨基酸没变 |
| 稀有 | 该密码子的相对适应度低于 CFD 阈值 |
| 非同义/错误位点 | 该密码子翻译后不是目标氨基酸 |

如果出现“非同义/错误位点”，说明该候选不是单纯的密码子优化，应回查序列来源。

## 7. 逐位点明细表是什么意思？

“逐位点明细表（导出用）”是对照图背后的长表。

每一行对应：

```text
某一条候选序列 × 某一个密码子位置
```

常用字段说明：

| 字段 | 解释 |
|---|---|
| `position` | 密码子位置，从 1 开始 |
| `amino_acid` | 目标蛋白该位置的氨基酸；终止密码子为 `*` |
| `candidate` | 候选名称 |
| `source` | 来源，例如 manual、IDT、GenScript、JCat、company_A |
| `codon` | 候选在该位置的密码子 |
| `reference_codon` | 原始 linB 在该位置的密码子 |
| `same_as_reference` | 是否与原始密码子完全相同 |
| `synonymous_with_target` | 是否翻译为目标氨基酸 |
| `host_frequency` | 该密码子在黑曲霉频率表中的频率 |
| `relative_adaptiveness` | 该密码子频率 / 同氨基酸最高频密码子频率 |
| `rare` | 是否低于 CFD 稀有阈值 |
| `gc3` | 密码子第三位是否为 G 或 C |
| `included_in_ranking` | 是否参与外部候选排序 |
| `is_local_reference` | 是否为程序本地生成的参考候选 |

页面只预览前 5000 行。需要完整数据时，请导出 CSV。

## 9. 推荐工作流程

### 场景 A：比较公司给的多个优化版本

1. 原始/目标序列中放原始 linB CDS。
2. 候选序列中放公司返回的多条 FASTA。
3. 点击分析。
4. 先看是否全部通过验证。
5. 再看 `CSI`、`GC%`、`CFD%`。
6. 导出指标 CSV，作为讨论记录。

### 场景 B：比较 IDT / GenScript / JCat / 手动优化结果

1. 把不同来源序列都整理成 FASTA。
2. FASTA header 中写清来源，例如：

```text
>linB_IDT source=IDT
>linB_GenScript source=GenScript
>linB_JCat source=JCat
>linB_manual_v1 source=manual
```

3. 粘贴到候选序列框。
4. 点击分析。
5. 只比较通过验证的候选。

### 场景 C：想要一个本地参考序列

1. 只输入原始 linB CDS。
2. 保持勾选“生成程序参考候选”。
3. 点击分析。
4. 在页面底部查看三条本地参考候选。
5. 如需保存，导出 `aniger_local_reference_candidates.fasta`。

## 10. 常见问题

### Q1：为什么我导入的候选没有进入排序？

通常是因为没有通过翻译一致性验证。请看“未通过的外部候选”中的 `errors`。

### Q2：没有终止密码子可以吗？

作为完整 CDS 候选，建议必须有终止密码子。本工具会把外部候选缺 stop 视为错误，避免下游表达构建出问题。

### Q3：本地生成候选为什么不参与排序？

因为本地候选是程序根据频率表生成的参考，不是湿实验人员、网站或公司给出的真实备选方案。为了避免误导，它们只放在结果底部供参考。

### Q4：CSI 最高的一条一定最好吗？

不一定。表达量还受启动子、mRNA 结构、5' 区域、剪接/隐性调控元件、蛋白折叠、宿主状态、培养条件等影响。CSI 只是密码子适配指标。

### Q5：可以自动调用 IDT / GenScript / JCat 吗？

当前简化版不自动联网，也不保存账号或 Token。推荐从这些网站导出 FASTA / CSV 后导入本页比较。

### Q6：能直接拿本页结果下单吗？

可以把通过验证的 FASTA 作为候选材料，但下单前仍建议人工确认：

- 序列名称和来源是否正确。
- 是否需要添加酶切位点、连接臂、标签、信号肽等构建设计元素。
- 是否符合公司对 GC、重复序列、合成难度的要求。

## 11. 最小检查清单

每次决定采用某条候选前，至少确认：

- 候选在“外部/人工候选排序”中，而不是“未通过”中。
- `errors` 为空。
- `included_in_ranking = true`。
- 翻译蛋白与目标一致。
- `CSI`、`GC%`、`CFD%` 没有明显异常。
- 已导出并保存指标 CSV 和候选 FASTA。
